AWS Single Sign-On Setup Guide

Step 1: Sign in to AWS SSO Console

1. Go to the AWS Management Console: https://console.aws.amazon.com/.

2. Navigate to the "AWS Single Sign-On" service.

Step 2: Set Up AWS SSO

1. In the AWS SSO console, choose "Enable AWS SSO."

2. Choose a region for AWS SSO to store its data.

3. Click on the "Enable AWS SSO" button.

Step 3: Create a Directory

1. In the AWS SSO console, select "Directories" from the left navigation pane.

2. Choose "Add directory" and select the type of directory you want to add (AWS Managed Microsoft AD or an external identity provider).

3. Follow the prompts to complete the directory setup.

Step 4: Invite Users

1. In the AWS SSO console, go to "Users" from the left navigation pane.

2. Choose "Invite a user" and enter the user's email address.

3. Send the invitation to the user.

Step 5: Assign Permissions

1. In the AWS SSO console, go to "Assign users" from the left navigation pane.

2. Select the users you want to assign permissions to.

3. Choose the AWS accounts and applications you want to grant access to.

Step 6: Configure Applications

1. In the AWS SSO console, go to "Applications" from the left navigation pane.

2. Choose "Add a new application."

3. Follow the instructions to configure the application for single sign-on.

Step 7: Sign In Using AWS SSO

1. Users can sign in to the AWS SSO User Portal: https:///user/.

2. They will use their assigned credentials (username and password) to access the portal.

3. From the portal, they can access the assigned AWS accounts and applications.

Step 8: Monitor and Manage

1. Use the AWS SSO console to monitor user activity, manage permissions, and make changes as needed.

For more details, refer to the AWS SSO Documentation.
